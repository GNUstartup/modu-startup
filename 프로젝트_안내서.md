# 모두의 창업 지원 시스템 — 프로젝트 안내서

이 문서는 프로젝트의 개발·운영 규칙을 정리합니다.
새로운 기능을 추가하거나 기존 코드를 수정할 때 반드시 참고하세요.

## [UI 규칙] 확인·알림창 처리 방식

**확인·알림창은 브라우저 기본 `alert()` / `confirm()`을 쓰지 말고,
항상 사이트 디자인에 맞는 커스텀 모달(팝업)로 만든다.**

- 기존 모달들과 같은 스타일을 따른다:
  - 둥근 모서리 (`rounded-2xl`)
  - 그림자 (`shadow-xl`)
  - 배경 어둡게 (`bg-black/40 backdrop-blur-sm`)
  - 흰 배경 카드 (`bg-white`)
- 재사용 가능한 공통 확인 모달 컴포넌트가 있으면 그것을 사용한다.
- 확인 모달의 z-index는 그 아래 열려 있는 모달보다 높게 설정한다 (예: 상세 모달 `z-50` 위에 확인 모달 `z-[80]`).
- 단순 안내(정보 메시지)도 브라우저 `alert()` 대신 커스텀 안내 모달을 사용한다.

### 근거
브라우저 기본 `alert()` / `confirm()`은 사이트 디자인과 어울리지 않고,
브라우저마다 생김새가 달라 일관된 UX를 제공하지 못한다.

## 상태 흐름 (7단계)

| 단계 | 상태명 | 설명 |
|---|---|---|
| 1 | 담당자 검토 대기 중 | 참가자가 신청 완료, 관리자 검토 전 |
| 2 | 수정 필요 | 관리자가 1차 반려, 참가자가 수정 후 재제출 필요 |
| 3 | 사전 승인 완료 | 관리자가 1차 승인, 참가자가 증빙 제출 가능 |
| 4 | 정산 신청 | 참가자가 증빙 서류 제출 완료 |
| 5 | 정산 반려 | 관리자가 증빙 검토 후 반려 |
| 6 | 정산 중 | 관리자가 증빙 승인, 송금 진행 중 |
| 7 | 정산 완료 | 송금 완료, 최종 처리 종료 |

---

## API 규칙

- 상태 변경은 반드시 `apiUpdateStatus(params)` 를 사용한다.
  - `params`: `{ 신청번호, 상태, 반려사유?, 처리자? }`
  - 처리이력은 이 함수가 자동으로 기록한다.
- 컬럼명(한국어)은 절대 영어로 바꾸지 않는다.

## 기타 주의사항

- 기존 신청/승인/수정/정산/첨부/상태조정 흐름을 변경하는 작업은 전체 흐름을 먼저 확인한다.
- 새 기능은 기존 버튼·흐름을 건드리지 않는 보조 도구 형태로 추가한다.
- 빌드(`npm run build`) 통과를 확인한 뒤 커밋·푸시한다.
